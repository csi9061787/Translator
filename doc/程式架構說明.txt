1.版本號規則  @2023/09/14 Tim
	${專案序號}.${主程式序號}.${功能序號}.${小更新序號}
	- 專案序號每換一個專案就+1
	- 主程式序號為進到一個完全無關的功能+1
	- 功能序號為同一功能新增需求+1
	- 小更新為除錯或優化+1
	
2.路徑架構  @2023/09/14 Tim
	- appClientModule #2.0以前舊架構的程式，會陸續轉移
	- src #程式
		- main #主程式
			- java #java程式
			- resources #靜態資源
		- test #測試
	- resource #文件
		- config #外部參數檔範例
		- doc #程式相關文件
3.程式架構
	1.Main			# 進入點，引入外部參數並判斷要執行哪一個功能
	2.Application	# 功能進入點引導程式至相關的UI或程式
	3.View			# UI畫面
	4.Listener		# 事件，處理前後端的互動以及前端的事件
	5.Controller	# 程式端的第一線，紀錄功能的主流程
	6.Service		# 服務，處理主流城中個步驟的詳細邏輯
